# VIZ-01 — Visualization Template Intelligence: Requirement Document

**Author:** Business Analyst Agent  
**Date:** 2026-06-26  
**Status:** Approved — ready for development  
**Approved by:** CEO Agent  

---

## 1. Overview

Fix the visualization template selector to use the LLM-generated `template_hint` from `visual_spec` as the primary selection signal, with keyword-based title matching as fallback only.

---

## 2. Problem Statement

`lib/templates/selector.ts:selectTemplate(title, position)` selects a template using keyword regex matching on the subtopic title. The `visual_spec.template_hint` generated by the LLM in `session-content-generator.ts` is stored in the database but never passed to `selectTemplate()`.

Result: when a subtopic's title does not contain the exact keywords the selector expects, the wrong template is chosen. For comparison content this means one subject appears (e.g. Claude Sonnet) while the others (Haiku, Opus) are silently dropped.

---

## 3. Functional Requirements

### FR-01 — Accept template_hint as primary signal

`selectTemplate()` must accept an optional third parameter `templateHint?: string`.

When `templateHint` is provided and is a valid `TemplateName`, use it immediately — do not run keyword matching. Only fall through to keyword rules when `templateHint` is absent, empty, or not in the valid set.

```typescript
// Current signature:
export function selectTemplate(
  subtopicTitle: string,
  position: 'first' | 'middle' | 'last'
): TemplateName

// New signature:
export function selectTemplate(
  subtopicTitle: string,
  position: 'first' | 'middle' | 'last',
  templateHint?: string
): TemplateName
```

Logic order inside the function:
1. If `position === 'first'` → return `'TopicHero'` (unchanged, always wins)
2. If `position === 'last'` → return `'KeyTakeaway'` (unchanged, always wins)
3. If `templateHint` is a valid `TemplateName` → return it
4. Keyword matching on title (existing rules, unchanged)
5. Default → `'DefinitionTriptych'` (unchanged)

**Valid set:** all members of `TemplateName` union from `lib/templates/types.ts`. Build a `Set<string>` from the array at module level — do not hardcode.

### FR-02 — Pass hint from session-plan.ts

In `lib/session-plan.ts`, where `selectTemplate(sub.title, position)` is called, replace with `selectTemplate(sub.title, position, sub.visual_spec?.template_hint)`.

The `sub` object at that point comes from the stored subtopic outline which contains `visual_spec`. Verify the type has `visual_spec?.template_hint: string | undefined` — add it to the type if missing.

### FR-03 — Pass hint from generate-content route

In `app/api/sessions/[id]/generate-content/route.ts`, wherever `selectTemplate()` is called, pass the `template_hint` from the subtopic's stored `visual_spec` as the third argument.

Pattern to find: `selectTemplate(` — inspect each call site and pass the hint from the subtopic's `visual_spec.template_hint`.

### FR-04 — Extend comparison keywords (belt-and-suspenders)

Even with FR-01, add a model-name safeguard to the comparison rule in `selector.ts`. This catches cases where `template_hint` is absent but the title clearly lists models or tools:

```typescript
// Existing:
if (/\bvs\b|versus|compare|difference between|top tools|tools compared|platforms compared|platform comparison|tool landscape/.test(t)) {

// New (add to the regex):
if (/\bvs\b|versus|compare|difference between|top tools|tools compared|platforms compared|platform comparison|tool landscape|sonnet.*haiku|haiku.*opus|model tiers|model comparison|claude model/.test(t)) {
```

More generally, if a title lists 2+ proper nouns separated by commas or slashes, treat as comparison. Add: `/[a-z]+ (vs|or|and) [a-z]+ (vs|or|and) [a-z]+/.test(t)` before the default.

### FR-05 — No change to `selectTemplatesForSubtopics`

This function maps title arrays to templates. It does not have access to `visual_spec` at call time. Leave it unchanged — it is used in a different code path (KB generation) where `template_hint` is unavailable.

## 4. Non-Functional Requirements

- No new LLM calls
- No DB migration
- No change to any template data schema
- `npx tsc --noEmit` must pass after changes
- All existing template keyword rules remain intact — this is additive, not a rewrite

---

## 7. Acceptance Criteria

- AC-01: A subtopic with `template_hint: 'ComparisonTable'` renders a `ComparisonTable` regardless of its title
- AC-02: A subtopic with no `template_hint` still uses the keyword matching rules correctly
- AC-03: `selectTemplatesForSubtopics` behaves identically to before (no regression)
- AC-04: A title containing "Sonnet" or "model tiers" without "vs" correctly resolves to `ComparisonTable`
- AC-05: `npx tsc --noEmit` outputs zero errors

---

## 8. Edge Cases

| Case | Expected behaviour |
|---|---|
| `templateHint = 'TopicHero'` for a middle section | Honour the hint — TopicHero renders |
| `templateHint = 'invalid_string'` | Fall through to keyword matching |
| `templateHint = undefined` | Fall through to keyword matching |
| `position = 'first'` with any hint | Always `TopicHero` (position overrides hint) |
| `position = 'last'` with any hint | Always `KeyTakeaway` (position overrides hint) |
